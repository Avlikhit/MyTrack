# MyTrack.Api

> Presentation layer for the MyTrack application.

---

# Overview

MyTrack.Api is the entry point of the MyTrack application.

It exposes RESTful APIs that allow clients (Angular, mobile applications, or other consumers) to interact with the MyTrack system.

The API is responsible for receiving HTTP requests, validating input, invoking business services, and returning standardized responses.

Business logic should **never** be implemented directly inside controllers.

---

# Responsibilities

The API project is responsible for:

- Exposing REST endpoints
- Receiving HTTP requests
- Returning HTTP responses
- Input validation
- Authentication & Authorization
- Dependency Injection configuration
- Middleware configuration
- API documentation (Swagger/OpenAPI)
- Exception handling
- Request logging
- Response formatting

# Architecture

```
Browser / Angular
        │
        ▼
HTTP Request
        │
        ▼
Controller
        │
        ▼
Application Services
        │
        ▼
Infrastructure
        │
        ▼
Database
```

The API project **does not contain business rules**.

Business logic belongs to the **Application** layer.

---

The API project depends on:

# Development Guidelines

Controllers should:

- Be thin
- Contain no business logic
- Return appropriate HTTP status codes
- Use dependency injection
- Call services instead of repositories directly
- Validate incoming requests
- Return standardized responses
